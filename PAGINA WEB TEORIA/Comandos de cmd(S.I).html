COMANDOS
NOMBRE EXPLICACIÓN EJEMPLOS / ATRIBUTOS / SINTAXIS
DIR Muestra la lista de subdirectorios y archivos de un directorio.
  • DIR /P: vuelca en pantalla solo los directorios/archivos que caben en la misma, esperando a que el usuario pulse una tecla para continuar. (lo muestra por páginas)
  • DIR /W: visualiza únicamente el nombre del archivo o directorio, sin datos adicionales. Los directorios se muestran encerrados entre corchetes “[ ]”. (sólo muestra el nombre)
  • DIR nombre_del_archivo.extensión para ver si un archivo concreto se encuentra en el directorio. Por ejemplo: dir setup.exe.
  • DIR /L: lista con minúsculas.
  • DIR /S: lista el contenido de los subdirectorios.
  • DIR /B: lista solo el nombre de los archivos y directorios
  • DIR /O: ordena los ficheros con respecto a un orden. Si entre los dos puntos y la letra de la ordenación ponemos un guión “-” el orden se invierte.
    • N: por nombre en orden alfabético.
    • E: por extensión en orden alfabético.
    • S: por tamaño, de menor a mayor.
    • D: por fecha y hora. Del más antiguo al más moderno.
    • G: primero aparecen los directorios
  • DIR /A: muestra archivos con un atributo específico.
  • "help dir" para más opciones
CHDIR Muestra el nombre del directorio actual o cambia de directorio
MKDIR Crea un directorio
RMDIR Quita (elimina) un directorio. Si el directorio no está vacío tendremos que borrar previamente los archivos y subdirectorios que contiene o invocarlo con la opción /S.
TREE Muestra de forma gráfica la estructura de carpetas de una unidad o ruta. Sintaxis: TREE [unidad:][ruta][opciones]
COPY Copia uno o más archivos en otra ubicación. Sintaxis: copy[unidad:][ruta]copia
COPY CON El ordenador se queda esperando a que introduzcamos texto. Escribimos lo que queramos. Para finalizar, pulsamos F6 o CONTROL + Z (que es la marca fin de fichero) y a continuación pulsamos Intro. Sintaxis: COPY CON nombre_archivo y pulsamos Intro.
MOVE Mueve UNO o MÁS archivos y cambia el nombre a archivos y directorios.
  • Sintaxis: move [unidad:][ruta]origen ... [unidad:][ruta]destino.
  • Por ejemplo: move gladiator peliculas\accion
REN o RENAME Cambia el nombre de uno o más archivos.
  • Sintaxis: REN [unidad:][camino] NombreArchivo NombreNuevo
  • Ejemplo:
    • ren mago.txt magia.txt
    • ren dos\cd\adios hola
“DEL” y “ERASE” Elimina uno o más archivos. Si en vez del nombre de un fichero especificamos el nombre de un directorio, lo que hace es borrar los ficheros que contiene.
  • Sintaxis:
    • DEL nombreArchivo.extension
    • ERASE nombreArchivo.extension
  • Ejemplos
    • DEL *.EXE: Borra todos los archivos de extensión .EXE en el directorio actual.
    • DEL *.*: Borra todos los archivos del directorio actual (con o sin extensión). El sistema nos preguntará si estamos seguros (es equivalente a escribir DEL .).
    • DEL *.: Borra todos los archivos sin extensión.
    • ERASE hola.txt: Borra los ficheros “hola.txt” y “foto.jpg”.foto.jpg
    • ERASE /P hola.txt: Borra el fichero pidiendo confirmación.
    • DEL /A: R *.*: Borra todos los ficheros de sólo lectura en el directorio actual
    • DEL ventas /S: Borra todos los ficheros en el directorio “ventas” y subdirectorios
TYPE Muestra el contenido de uno o más archivos de texto. Para más de un fichero escribiendo los nombres separados por un espacioen blanco o usando comodines. Sintaxis: TYPE [unidad:][ruta]archivo
MORE Muestra la información pantalla a pantalla. More es una mejora de "TYPE" ya que hace pausas si el contenido mostrado es muy largo Sintaxis: MORE [opciones][unidad:][ruta] archivo
SORT recibe como entrada un conjunto de caracteres organizados en filas y devuelve los mismos caracteres ordenando cada fila alfabéticamente.

Sintaxis: SORT [opciones][unidad:][ruta] archivo

  • alfabéticamente: sort lista.txt
  • Orden inverso: sort
  • 3º Carácter: sort/+3 lista.txt
REDIRECCIONES Para variar la entrada de una orden, la salida o ambas se utilizan los operadores de redireccionamiento.
  1. Salida
    • Ejemplo: type cosas.txt > mas.txt.
    • Explicación: Type muestra el contenido de "cosas.txt" y la salida es en "mas.txt"
  2. Entrada
    • El operador de entrada (“<“) toma los datos de entrada desde el fichero o dispositivo especificado.
    • En el siguiente ejemplo: MORE < "fich1.txt". MORE hace que se muestra de página a página. El contenido del fichero “fich1.txt”. Se muestra por pantalla
  3. Adicción
    • El operador de adicción a un fichero (“>>”) dirige la ejecución de un comando a un fichero. Si el fichero existe añade el resultado de la ejecución al final del fichero sin sobrescribir lo anterior. Si no existe se crea.
  4. Tubería
    • En el siguiente ejemplo: DIR | SORT /R. Redirige la salida del comando “dir” al comando “sort”
FIND Busca una cadena de texto en uno o más archivos.
  • Sintaxis: FIND [opciones] "cadena" [[unidad:][nombre de ruta de acceso]...]
  • Ejemplo: FIND “muestra en” ayuda.txt
  • Busca el texto "muestra en" en el archivo “ayuda.txt”
FINDSTR Busca cadenas en los archivos. Dispone de una amplia colección de comodines que permiten construir un gran número de expresiones regulares para afinar la búsqueda.
  • Sintaxis: FINDSTR [opciones] "cadenas" [[unidad:][ruta de acceso]...]
    • /b: Hace coincidir los modelos si están al principio de la línea.
    • /c:cadena: Usa una cadena especificada como una búsqueda de cadena literal.
    • /i: Especifica que la búsqueda no distingue mayúsculas de minúsculas.
    • /s: Busca archivos que coinciden en el directorio actual y en todos los subdirectorios.
    • /f:archivo: Lee la lista de archivos desde el archivo especificado (/ significa consola)
    • /g:archivo: Toma la búsqueda de archivos desde el archivo especificado(/ significa consola).
    • /m: Solo imprime el nombre de archivo si el archivo contiene una correspondencia
FC Compara dos archivos o conjuntos de archivos y muestra las diferencias entre ellos. (La comparación puede ser binaria o en modo texto y, en modo texto, puede usarse el código ASCII o el UNICODE.)
  • Sintaxis: FC [opciones] [unidad1:] [camino1] NombreArchivo1 [unidad2:] [camino2] NombreArchivo2
    • /c: Omite mayúsculas y minúsculas.
    • /n: Muestra los números de línea en una comparación ASCII.
    • /b: Ejecuta una comparación binaria.
  • Ejemplos
    • FC /c fichero1 fichero2: Ignorar en la comparación mayúsculas y minúsculas. (/c)
    • FC /n fichero1 fichero2: Mostrar el número de línea durante la comparación (/n)
    • FC /b c fichero1 fichero2: Comparación binaria de dos archivos (/b)
    • FC /n *.txt archivo: Comparar todos los archivos de extensión txt del directorio con oro archivo. (/n)
ATTRIB Muestra o cambia los atributos de un archivo.
  • Sintaxis: ATTRIB [opciones] [unidad :] [ruta] [nombrearchivo]
  • Ejemplos de atributos son:
    • R: atributo de archivo de sólo lectura.
    • S: atributo de archivo de sistema.
    • H: atributo de archivo oculto.
    • A: atributo de archivo de almacenamiento.
  • Ejemplos de “ATTRIB”
    • ATTRIB –R: Quitar el atributo de sólo lectura al fichero “protegido.txt”.
    • ATTRIB otros: Consultar los atributos del fichero “otros”.
    • ATTRIB +r +h otros: Ocultar el fichero “otros” y hacerlo de solo lectura
    • ATTRIB /s +R *.txt: Establecer el atributo de “solo lectura” en el directorio actual y sus subdirectorios para todos los ficheros de extensión “txt”.
PROMPT Cambia el símbolo del sistema de cmd.exe. Cuando abrimos una ventana del intérprete de comandos, el texto que se muestra antes del cursor parpadeando recibe el nombre de prompt
  • PROMPT $d: Mostrar como prompt la fecha actual.
  • PROMPT $t: Mostrar como prompt la hora actual.
  • PROMPT $t $v: Mostrar como prompt la hora actual y la versión de Windows.
  • PROMTP $: Hacer que sólo se muestre el cursor parpadeando.
